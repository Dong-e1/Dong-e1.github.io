Gulp搭建的小米静态网页

发表于2021-06-02|更新于2021-06-02|项目小米商城
|字数总计:977|阅读时长:3分钟|阅读量:

小米商城静态网页

这个静态网页是在学习完JavaScript基础时，想去了解整个网页体系，然后去找资源尝试练习。

页面功能也很简单，主要是练习了数据的请求获取以及对json数据的处理，轮播图，购物车加购等功能

前期准备

前端技术：HTML+CSS+JavaScript+jQuery

开发工具：Gulp，Sass，require.js

Gulp

前端自动化系统构建工具

  1. 用npm或者cnpm安装gulp包 npm install gulp -g

  2. 关于gulpfile.js文件的一些配置

    目录下新建 gulpfile.js文件——gulp项目配置文件

    安装 第三方插件 `gulp-sass` `gulp-minify-css` `gulp-rename`

    gulpfile.js文件里引入 gulp和插件

工作流程 把.scss文件=>css文件=>压缩=>min.css

使用task()时,每个任务都会接受一个回调函数做为参数，不处理的话，就会报错

gulp: Did you forget to signal async completion

分别建立 任务
处理文件任务 “scss”, “scripts”, “copy-html”, “data”, “scssAll”, “images”

任务 build => 一次性执行多个任务

任务 watch =>建立监听文件改动，并进行文件任务操作

启动 一个服务器 npm安装`gulp-connect`

启动一个默认任务 直接使用gulp运行

gulp.task("default", ["watch", "server"]);

require.js

页面引入

<script src='js/require.js' defer async='true' data-main='js/main.js'></script>

  1. 所有js必须通过模块化开发实现 require.js AMD规范

  2. 一切文件路径都以dist文件路径为准

  3. 加载 requirejs 脚本的 script 标签加入了data-main属性 使每一个页面都使用这个配置

  4. defer,async=’true’是浏览器异步加载

  • 如果async为true，那么脚本在下载完成后异步执行。
  • 如果async为false，defer为true，那么脚本会在页面解析完毕之后执行。
  • 如果async和defer都为false，那么脚本会在页面解析中，停止页面解析，立刻下载并且执行。

main.js

  1. 是全局配置文件 配置当前这个项目用到了哪些模块

  2. 遵从的都是AMD规范

  3. 所有.js文件，后缀都可以省略

导入其他模块

方法：require()

语法：require([依赖文件1，依赖文件2，…]，function(模块1，模块2，…){})

nav.js

  1. 处理首页导航部分

  2. 声明模块遵从AMD

  3. 依赖其他模块的模块

使用define()定义

语法：define([依赖文件1，依赖文件2，…]，function(模块A，模块B，…){})

利用jQuery中的ajax语法，获取数据

利用return，向外暴露内容。用于其他模块引用
